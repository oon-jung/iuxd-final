## 1. 프로젝트 개요

### 1.1 작품
영화 「왕과 사는 남자」 — 폐위된 어린 왕 **단종**(1441–1457)과 영월의 주민 **엄흥도**(嚴興道)의 실제 역사에서 출발한 작품. 관객 1,500만 명 이상.

### 1.2 과제 목표
후속 경험(팝업 전시·체험 콘텐츠)의 부재를 해결하고, **영화의 감정을 웹 기반 인터랙티브 전시로 확장**한다. 과제 규정상 기술 완성도가 아니라 **UX 설계의 논리성과 경험 전달력**을 평가 대상으로 한다.

### 1.3 핵심 메시지

> **"당신의 단종은 누구인가요?"**
>
> 엄흥도에게 단종이 그러했듯, 당신에게도 지켜주고 싶은 존재가 있습니까.

## 2. 문제 정의

### 2.1 v1.0 평가 요약

| 항목 | 비중 | 점수 |
|---|---|---|
| 사용자 조사 및 인사이트 | 25% | 7/10 |
| UX 설계 완성도 | 30% | 9/12 |
| 인터랙션 창의성 | 25% | 9/10 |
| 프로토타입 구현 | 20% | 6/8 |
| **합계** | 100 | **31/40** |

**평가자 지적 3대 약점**
1. **설문 분석 깊이 부족** — 단순 비율 수준
2. **UX flow 논리 연결 약함** — 기승전결이 유기적으로 이어지지 않음
3. **인터랙션 차별성 부족** — 범용 패턴만 사용

### 2.2 v1.0의 핵심 결함 — 은유의 자의성
*"왜 단종이 고양이로 비유되는가?"* 에 답하는 구간이 전무했다. 관객은 영상 2(독약 씬) 이후에야 비유를 인지하며, 마지막 질문이 돌연하게 느껴진다.

### 2.3 v3.0 방향 선언
> 전시 전반부에 **"왜 고양이인가"를 논리적으로 설명하는 6페이지 구간**을 삽입하고,
> 각 섹션 전환에 **서사적 필연성**을 부여하며,
> 본 전시 주제에 특화된 **5종 시그니처 인터랙션**을 배치한다.

## 3. 사용자 조사 및 인사이트

### 3.1 설문 개요
- **대상** : 20·30대 30명 (구글 폼 5개 섹션)
- **목적** : 몰입형 콘텐츠 기대치 · 체험 요소 선호도 측정

### 3.2 비율 → 인사이트 3단 분석

| 응답 | 비율 | **숨은 이유 (Why)** | **본 전시 반영** |
|---|---|---|---|
| 감정 몰입 | **52.9%** | 정보 과잉이 몰입을 방해한다. 여백·정적이 필요. | Page 14·17·20 암전형 독백 · Page 18 3D 사진 정적 응축 |
| 체험 요소 | **26.5%** | 관객은 당사자가 되길 원한다. | Page 4 관계 탐방 · Page 11 피딩 · Page 15 드래그 |
| 주요 씬 재경험 | **41.2%** | 감상의 잔상이 반복 접근의 동기. | Page 18 추억 3D · Page 22 강물 편지 |
| 스토리 선택 변화 | **11.8%** | 소수지만 재방문·공유의 동인. | Page 4 클릭 분기 · Page 15 드래그 해석 |
| 시각 완성도 | **6.6%** | 화려함보다 진정성. | 흑백 기조 + 고문서 톤 |

### 3.3 자유응답 키워드 (N=24 유효 응답, 상위 8)

| 순위 | 키워드 | 빈도 | 해석 |
|---|---|---|---|
| 1 | 여운·잔상 | 54% | 능동적 감정 체류 공간 필요 |
| 2 | 허무·무력감 | 38% | 카타르시스보다 여백 |
| 3 | 짠함·가엾음 | 29% | 약자 서사 공감 준비됨 |
| 4 | 정적·침묵 | 25% | 사운드 부재 구간 근거 |
| 5 | 긴장·불안 | 21% | 위기 섹션 연출 근거 |
| 6 | 체험 욕구 | 21% | 관찰자 지위 거부 |
| 7 | **"나도 지켜주고 싶다"** | **17%** | **은유 수용성 사전 확인** |
| 8 | "역사가 가깝다" | 13% | 재현 아닌 재해석 선호 |

**결정적 발견** — 7번 키워드 *"나도 지켜주고 싶다"* 가 17%. 관객은 이미 **"지켜주고 싶은 존재"** 심리 장치에 공감할 준비가 되어 있다. 본 전시는 그 대상을 **단종 ↔ 반려동물**로 연결하는 논리 브릿지만 제공하면 된다.

## 5. UX 컨셉 · 핵심 논리

### 5.1 "왜 고양이인가?" 논리 체인 (v3 신설)

```
엄흥도가 단종에게 품은 감정
       │
       ├─ 충성 (왕을 향한 신하의 마음)
       └─ 부성애 (어린 생명을 거두는 어른의 마음)  ★핵심
              │
              ▼
   타깃 세대(20·30대)가 부성애·모성애를
     가장 가까이 느끼는 대상
              │
              ▼
         반려동물 (특히 반려묘)
              │
              ▼
∴ 현대의 엄흥도 = 반려묘의 주인
   반려묘 = 현대의 단종
```

### 5.2 타락과 구원의 구조 (v3 보강)
본 전시의 주인공은 **"순수한 선인"이 아니다.** SNS 인플루언서의 허영·돈벌이 동기로 고양이를 들이기 시작해, 시간이 흐르며 부모의 감정을 발견한다. 이 **이중 구조**가 은유의 깊이를 만든다.

| 영화 속 엄흥도 | 본 전시의 현대 주인공 |
|---|---|
| 처음부터 충절을 가진 자 (전통 서사) | SNS 수익·트렌드에 편승한 자 (현대 출발점) |
| 단종을 지키다 목숨을 걺 | 고양이가 죽은 후에야 자신의 마음을 발견 |
| → **부성애의 완성** | → **부성애의 각성** |

이 대칭이 **현대 관객에게 영화 속 엄흥도를 '자신과 다른 인물'이 아닌 '각성 후의 자신'으로 느끼게** 한다.

### 5.3 설계 원칙
1. **Early anchoring** — 은유의 연결고리를 전시 30% 이내에 명시화
2. **Layered explanation** — 영화 미관람자도 3문장으로 수용
3. **Emotional breathing** — 감정 장면 사이 여백 (정적·암전)
4. **Participatory agency** — 사용자 선택이 공감 깊이를 조절
5. **Redemption arc** — 주인공의 이기적 출발이 타깃과 동기화, 이후 성장

### 5.4 벤치마크 대비 차별점

| 유사 전시 | 몰입 방식 | 본 전시 차이 |
|---|---|---|
| D뮤지엄 「I Love Paper」 | 촉각·공간 | 물리를 마우스/스크롤로 번역 |
| 롯데월드 플로럴 가든 | 시각·조명 | 서사 추가 |
| 국립중앙박물관 사유의 방 | 정적·조도 | 부재의 인터랙션을 웹에 구현 |
| **The Cat's Warden** | **서사 × 감정** | **역사 + 현대 · 은유 · 각성 구조** |

## 7. 시그니처 인터랙션 (6종)

### IX1 — Relationship Gallery Scatter (Page 4) ★핵심
- **메커니즘** : 6개 폴라로이드(부모자식·친구·연인·**반려동물**·조부모·동료)가 화면에 흩어짐. 마우스 호버 시 카드가 커서 방향으로 기울며 확대. 반려동물 카드만 은은한 발광 + 2.6s 펄스 애니메이션 + ring 링 애니메이션. 비-반려동물 카드 클릭 시 흔들림(500ms), 반려동물 카드 클릭 시 중앙으로 확대(1.2s) 후 전체 화면 확장으로 *"오늘의 엄흥도는 누군가의 반려인입니다"* 발화.
- **레퍼런스** : 폴라로이드 스캐터 배치, 각 카드 `--tx/--ty/--rot` CSS 변수로 랜덤 흩뿌림
- **차별성** : 클릭 이동이 아닌 **시각적 힌트에 의한 논리 유도**. 관객이 스스로 "반려동물이 답"임을 깨닫게 한다.

## 9. 프로토타입 사양

### 9.1 기술 스택

| 레이어 | 기술 |
|---|---|
| 프레임워크 | React 19 |
| 애니메이션 | GSAP + CSS animations |
| 스크롤 감지 | IntersectionObserver |
| 3D 모션 | CSS 3D transform (Page 18, 22) |
| 영상 | HTML5 `<video>`, preload on-demand |
| 로컬 저장 | localStorage (Page 22 이전 편지 누적) |
| 라우팅 | HashRouter (GitHub Pages 호환) |

### 9.2 배포

- **배포 대상** : `https://oon-jung.github.io/IUXD_Final/`
- **빌드** : `npm run build && npm run deploy` (gh-pages 브랜치 자동 푸시)
- **`package.json`의 `homepage`** 필드를 배포 URL로 업데이트 필요

### 9.3 접근성

- `prefers-reduced-motion` 지원 (모든 신규 컴포넌트)
- 키보드 네비게이션 (Enter/Space/Arrow)
- 모바일 터치 대응
- 스크린리더 aria-label
- BGM 토글 (예정)

### 9.4 성능 목표

| 지표 | 목표 | 비고 |
|---|---|---|
| Lighthouse Performance | 90+ | 이미지 lazy-load |
| 첫 화면 로드 | < 2s | critical path 최적화 |
| 페이지 전환 | < 300ms | 달성 |
| 총 에셋 | < 350MB | video3 491MB → 압축 고려 |
