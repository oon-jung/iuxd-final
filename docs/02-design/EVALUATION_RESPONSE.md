### 2-1. "설문 분석 깊이 부족 (단순 비율 수준)"

**현재 기획서 (PPT p.5):**
- 두 파이차트만 제시 (What people think important / What contents want)
- "People value Experiential Element... want to change stories through choices" 한 줄 인사이트

**문제:**
- 비율만 있고 **"왜"**가 없음
- 페르소나-설문 응답의 교차 분석 없음
- 자유응답(섹션 5) 분석 누락
- 디자인 의사결정과 설문 데이터의 인과 링크 불명확

**개선안 (PPT에 추가할 섹션):**

#### A. 설문 설계 근거 페이지 신설
```
설문은 왜 이렇게 설계했는가?
├─ 섹션 1: 영화 관람 경험 → 기본 맥락 파악
├─ 섹션 2: 몰입형 콘텐츠 경험 → 기대치 파악
├─ 섹션 3: "가볼 만한데 경험 없는 곳" 인식 → 유적지의 UX 공백 측정
├─ 섹션 4: 중요 가치 선택 → 디자인 우선순위 결정 근거
└─ 섹션 5: 자유응답 → 예상 밖 인사이트 발견
```

#### B. 인사이트 추출 3단 표
| 설문 응답 | 단순 비율 | **Insight (Why)** | **Design Decision** |
|---|---|---|---|
| 감정 몰입 52.9% | "사람들은 감정 몰입을 원한다" | 정보 과잉이 오히려 몰입을 방해, 감정 곡선 설계가 중요 | 섹션 간 BGM 레이어링, 빈 화면 여백 확보 |
| 체험 요소 26.5% | "체험도 중요" | 주도적 참여 = 기억에 남는 전시, 수동적 관람 거부 | 피딩 미니게임, 좋아요 인터랙션 |
| 스토리 선택 변화 11.8% | "작지만 존재" | 소수지만 재방문 동기 대표 그룹, P1 페르소나와 일치 | 분기 선택지 2개 포함 (S3 Village Notice) |
| 자유응답 키워드 TOP5 | 미분석 | "여운/허무/짠함/긴장/정적" → **정적 체험 공간 선호** | 사운드 디자인, 암전 전환, 2초 여백 |

#### C. 페르소나-설문 연결 다이어그램
```
Persona 1 (감정 몰입형) ──→ 감정 몰입 52.9% + 자유응답 "여운"
Persona 2 (라이트 팬)   ──→ 체험 요소 26.5% + 자유응답 "긴장"
Persona 3 (SNS 기록형)  ──→ 주요 씬 재경험 41.2% + 자유응답 "인증"
                              ↓
                     각 페르소나 → 전용 터치포인트 매핑
```

#### D. 경쟁/유사 사례 리서치 추가 (현재 전무)
- 롯데월드 "플로럴 가든" 몰입형 전시
- D뮤지엄 "I Love Paper" 인터랙션
- 국립중앙박물관 "사유의 방" 정적 몰입
- → 본 전시의 **차별점** 명시 ("정적 인내 인터랙션 + 은유 구조")

**평가 회복 기대:** 사용자 조사 및 인사이트 7 → 9점 (+20%)

### 2-2. "UX flow 논리 연결 약함"

**현재 기획서 (PPT p.10-13):**
- 기-승-전-결 각 단계에 Web Interaction / Video만 표기
- 단계 간 **왜 그렇게 연결되는지** 논리 링크 없음
- 영화 스토리와 전시 UX 매핑표(p.14)는 좋지만 일방향

**개선안:**

#### A. User Journey Map 신설 (감정 곡선 포함)
```
시간축    0s───5s───────30s──────60s─────120s──────180s
감정값  +1 ●                                    
        0  ────●───●        ●───                ●
       -1          ●────●       ●────●────●
            인트로 좋아요 빈공간 만남 피딩     위기    애도   기록
                   (쾌) (대기) (신뢰)  (유대)  (불안) (슬픔) (성찰)
```

#### B. 단계 간 논리 링크 명시
각 섹션 전환에 "Why this next?" 한 줄:

| 전 → 후 | Why (논리 링크) |
|---|---|
| Intro → S1 맥락 | "단종"이 누구인지 모르면 은유 성립 불가 |
| S1 → 좋아요 인터랙션 | 역사 서사 → 현대 SNS로 시대 점프, 대비 효과 |
| 좋아요 → Video 1 | "좋아요"는 동기, 실제 행동(귀가)은 영상으로 전환 |
| Video 1 → S2 만남 | 행동 이후 고양이와 **첫 조우** 감정 단계 |
| S2 → 피딩게임 | 신뢰 형성 → 유대 형성 (시간 경과) |
| 피딩 → S3 공고 | 사적 유대 ↔ 사회적 갈등의 대립 진입 |
| S3 → Video 2 | 선택의 결과 관찰 (관찰자 → 당사자) |
| Video 2 → S4 은유 | 현대 사건을 역사와 겹쳐 은유 명시화 |
| S4 → Video 3 | 은유 수용 후 애도 장면으로 **카타르시스** |
| Video 3 → 입력 | 감정 정점 → 개인 성찰로 승화 |
| 입력 → S5 추모벽 | 개인 기억 → 공동체 기억 확장 |

#### C. Decision Points (분기) 표시
- **Village Notice(S3)**: 서명/거부 선택 → 양쪽 모두 Video 2로 수렴하지만 엔딩 메시지 변주
- **BGM 토글**: 조용히 감상 vs 몰입 감상 선택
- **Memory Wall**: 익명 공유 허용 여부

**평가 회복 기대:** UX 설계 완성도 9 → 11점 (+17%)

### 2-3. "인터랙션 차별성 부족"

**현재 인터랙션:**
- 클릭 좋아요 (흔함)
- 드래그 피딩 (흔함)
- 스크롤 + Next 버튼 (흔함)

**문제:** 평균 수준의 웹 인터랙션 패턴만 사용. 영화/전시 맥락에 특화된 시그니처 인터랙션 부재.

**개선안 — 시그니처 인터랙션 4종:**

#### IX1. 「Patience Interaction」 (인내 인터랙션) — ★시그니처
- **위치**: S2 First Meeting
- **메커니즘**: 마우스를 **움직이면** 고양이가 도망, **멈춰 있으면** 다가옴
- **차별성**: 웹 관습 정반대 — 가만히 있는 것이 진척
- **은유**: 엄흥도가 단종에게 다가갈 때의 조심스러움
- **구현**: mousemove 이벤트 throttle 300ms, stillness timer

#### IX2. 「Collective Choice」 (집단의 선택) — 새로움
- **위치**: S3 Village Notice
- **메커니즘**: 선택 직후 "지금까지 방문자 중 X명(XX%)도 같은 선택을 했습니다" 실시간 표시
- **차별성**: 개인 선택이 집단 데이터로 환원, 관객이 수치의 일부가 되는 체험
- **은유**: 세조에 동조한 다수/거역한 소수 — 역사는 집단 선택의 총합
- **구현**: localStorage 간이 카운터, 또는 정적 샘플 데이터

#### IX3. 「Dual Exposure Scroll」 (이중 노출 스크롤) — 영화적
- **위치**: S4 Metaphor Bridge
- **메커니즘**: 스크롤 양에 따라 두 이미지(엄흥도/현대인)의 불투명도가 교차
- **차별성**: 스크롤 = 시간축 이동, 600년이 물리적으로 겹치는 체감
- **은유**: 역사와 현재는 같은 장면의 이중 노출
- **구현**: IntersectionObserver + scroll progress ratio

#### IX4. 「Silence as Climax」 (정적 클라이맥스) — 감정적
- **위치**: Video 3 직전 3초
- **메커니즘**: BGM이 **사라지는** 순간에 화면이 정지, 사용자 클릭이 있어야 다음 재생
- **차별성**: 사운드 절정이 아닌 **부재**가 감정 정점
- **은유**: 죽음의 적막
- **구현**: Web Audio API fade-out, 클릭 대기 모드

#### IX5. 「Memory Stars」 (기억의 별) — 잔상
- **위치**: S5 Memory Wall
- **메커니즘**: 내가 입력한 "단종"이 화면에 별로 떠오르고, 다른 방문자들의 별과 함께 천천히 움직임
- **차별성**: 내 흔적이 사라지지 않고 공동의 별자리가 됨
- **은유**: 애도는 개인이 아닌 공동의 기억
- **구현**: Canvas or CSS particles + localStorage

**평가 회복 기대:** 인터랙션 창의성 9 → 10점 (+10%)

## 6. 기획서와 웹 구현의 상호 강화

평가자가 **두 번 모두** "깊이 부족"을 지적 → 기획서와 구현이 **서로를 증거**로 삼아야 함.

| 기획서 주장 | 웹 구현 증거 |
|---|---|
| "감정 곡선 설계" | 섹션별 BGM 전환 실제 작동 |
| "인내 인터랙션" | S2 마우스 정지 감지 코드 |
| "집단 기억" | S5 localStorage 누적 데이터 |
| "은유 명시화" | S4 스크롤 블렌딩 실제 작동 |
| "맥락 안내" | S1 자동 타이포그래피 |

→ 발표 때 기획서 한 페이지 읽고 → 해당 웹 시연을 QR로 즉시 보여주는 **이중 증명 구조**.
