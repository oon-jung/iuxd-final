# 「The Cat's Warden」 개선 Plan

> 작성일: 2026-04-23
> 작성 배경: 기존 웹 디자인을 유지한 채 개연성 보완 슬라이드와 UX 흐름 개선 추가
> 프로젝트: 영화 「왕과 사는 남자」 기반 인터랙티브 전시 웹 (기·승·전·결 4막 구조)

## 1. 문제 진단 (Problem Diagnosis)

### 1-1. 핵심 문제: 은유의 개연성(Plausibility) 붕괴
현재 플로우는 "단종=고양이"라는 비유를 **관객의 추론에 과도하게 의존**한다.
관객이 은유를 깨닫는 지점이 **Video 2(위기 씬) 이후**로 너무 늦다.

| 단계 | 현재 관객이 느끼는 것 | 기획 의도 |
|---|---|---|
| 기 (Intro~Insta) | 귀여운 고양이 SNS 이야기 | 엄흥도의 "단종이 우리 마을에 왔으면" 욕망 |
| 승 (피딩게임) | 고양이 키우기 시뮬 | 엄흥도-단종의 신뢰 형성 |
| 전 (Video 2) | 갑자기 무거운 사건 | 세조의 단종 핍박 |
| 결 (Video 3, 입력) | "내 단종? 그게 뭐지?" | 지켜주고 싶었던 존재 회상 |

**관객의 은유 인식 시점이 전시의 80% 지점**에서 일어남 → 몰입 붕괴.

### 1-2. UX 흐름의 유기적 단절
- 섹션 간 전환이 **모두 수동 "Next" 버튼**으로 통일되어 감정 리듬 없음
- "SNS 수익 목적" → "진정한 애도"로 가는 **감정 곡선의 중간 기착점**이 부재
- 배경 음악/사운드 디자인 없음 → 감정 앵커 부족
- 관객이 자신의 선택/개입이 스토리를 바꾼다는 느낌이 약함 (설문 11.8%만이 '선택→변화' 체감 원함 → 역설적으로 이 부분이 약점)

### 1-3. 기획서 PPT 문제와의 연결
> 설문 내용이 현재 단종-엄홍도 ↔ 주인-반려묘 재해석을 **논리적으로 뒷받침하지 못함**

- 설문 결과 핵심: **감정 몰입 52.9%** / **체험 요소 26.5%** / **주요 씬 재경험 41.2%**
- 현재 전시는 "체험"은 있지만 "**영화의 감정 몰입을 확장**"하는 기능은 약함
- 해석 링크(movie → cat) 설명이 포스터 한 장으로만 암시됨

## 2. 설계 원칙 (Design Principles)

기존 웹디자인/코드베이스를 **유지**하면서 다음을 추가:

1. **Earlier anchoring**: 은유의 연결고리를 전시 초반 30% 안에 명시적으로 노출
2. **Layered explanation**: 영화를 모르는 관객도 기본 서사를 3문장으로 이해
3. **Emotional breathing room**: 감정 장면 사이 숨 고르기/여백 슬라이드
4. **Participatory agency**: 사용자 선택이 '공감의 깊이'를 조절하는 인터랙션
5. **Collective memory**: 이전 관람자 흔적(이미 입력된 "단종") 보여주기

## 3. 제안하는 신규 슬라이드 (New Slides)

> 번호는 *삽입 위치* 기준. 기존 12개 섹션 사이에 인서트.

### S1. 「When the King Came to the Village」 - 맥락 브릿지 (★필수)
- **위치**: Intro 포스터 애니메이션 직후, About 섹션 (about-ref) 앞
- **목적**: 영화 미관람 관객에게 기본 서사 3초 주입
- **형태**:
  - 검정 배경 + 흰색 활자로 3줄 타이포그래피 (auto-type 애니메이션, 총 6초)
  - "1457년, 폐위된 어린 왕이 강원도의 한 마을에 왔다."
  - "주민들은 그를 미워했지만, 한 사람은 매일 밥을 지어 가져갔다."
  - "그 사람의 이름은 엄흥도."
  - 하단: "지금도, 누군가는 그렇게 살고 있습니다." (fade-in)
- **인터랙션**: 6초 후 자동 진행, 하단 '↓ 계속 보기' 인디케이터만

### S2. 「The First Meeting」 - 첫 조우의 주저 (★필수)
- **위치**: Video 1(친구와 대화/귀가) 직후, CatFeedingGame 앞
- **목적**: "SNS 수익 목적"에서 "진짜 감정"으로 넘어가는 교량
- **형태**:
  - 어두운 골목 배경, 불 꺼진 거리. 고양이가 멀리서 응시
  - 사용자가 마우스를 고양이 쪽으로 움직이면 고양이가 뒷걸음
  - 3초 이상 마우스를 **멈추고 있으면** 고양이가 다가옴 (인내=신뢰)
  - 고양이가 화면 중앙까지 왔을 때 자막: "이 고양이는 당신이 기다려 준 만큼만 다가옵니다."
- **인터랙션**: 마우스 정지 감지(mousemove 이벤트 throttle), 3초 쿨다운
- **은유 연결**: 엄흥도가 단종을 관찰·접근하는 조심스러움 = 이 슬라이드

### S3. 「The Notice on the Wall」 - 사회적 압력 (★필수)
- **위치**: CatFeedingGame Day 30 완료 후, response3 섹션 대체 또는 앞 인서트
- **목적**: Video 2의 "주민의 포획" 장면이 갑자기 나오지 않도록 빌드업
- **형태**:
  - 벽에 붙은 종이 공고문 이미지 (스크롤로 확대되는 parallax)
  - 공고문 텍스트: "길고양이 개체수 감소 서명 운동 - 주민 123명 서명"
  - 하단 두 버튼: [서명한다] / [돌아선다]
    - 둘 다 같은 결과(Video 2)로 진행하지만, 선택이 Web Interaction 3의 최종 입력 화면에 기록됨
    - "당신은 그날 [돌아섰다]. 123명 중 한 명이 되지 않았다."
- **은유 연결**: 세조의 핍박·마을 주민의 동조 ↔ 현대의 '집단 민원'

### S4. 「Two Faces, One Grief」 - 명시적 은유 연결 (★필수)
- **위치**: Video 2(독약/위기) 직후, response4 대체 또는 앞 인서트
- **목적**: 단종=고양이 은유를 관객에게 **명시적으로** 드러내는 단 한 번의 moment
- **형태**:
  - 좌: 어둠 속 엄흥도가 아이를 찾는 실루엣 (영화 스틸컷)
  - 우: 골목에서 고양이를 찾는 현대인 실루엣
  - 두 이미지가 천천히 ← → 으로 겹쳐짐 (CSS blend-mode overlay 또는 opacity crossfade)
  - 중앙 텍스트 fade-in: "**600년의 시간이 두 얼굴을 포갭니다.**"
  - 아래 희미하게: "지켜주려는 자의 마음은, 시대를 건너 같습니다."
- **인터랙션**: 스크롤 진행도에 따라 두 이미지 겹침 정도 조절 (IntersectionObserver + scroll progress)
- **은유 연결**: 여기서 관객은 비로소 **"아, 이 고양이가 단종이었구나"** 깨달음

### S5. 「Others' Danjong」 - 집단 기억 (선택)
- **위치**: FinalSection(입력창) 제출 후, 엔드 크레딧 직전
- **목적**: 개인 경험을 공동체 기억으로 확장, 재방문 동기 생성
- **형태**:
  - 제출 버튼 클릭 → 화면이 검은 바탕의 '추모벽'으로 전환
  - 이전 관람자들이 입력한 "나의 단종" 텍스트가 별처럼 떠오름 (30~50개, 샘플/익명)
  - 사용자가 방금 입력한 단종은 중앙에 천천히 발광
  - 하단: "당신의 단종은 이제 많은 이들의 기억과 함께 있습니다."
- **인터랙션**: localStorage 저장 + 샘플 추모글 JSON
- **설문 연결**: 41.2% "주요 씬 재경험" / "기록 남기기" 페르소나3 니즈

### S6. 「Audio Cue Layer」 - 전 구간 사운드 디자인 (★필수)
- **위치**: 모든 섹션 배경
- **목적**: 감정 앵커 강화. 현재 **BGM 전무**가 가장 큰 몰입 저해 요소
- **형태**:
  - 3레이어 BGM:
    - L1 (기): 가벼운 SNS 알림음 + 현대 앰비언트
    - L2 (승): 빗소리/바람/새소리 + 약한 가야금 톤
    - L3 (전): 저주파 드론 + 바이올린 서스테인
    - L4 (결): 가야금 단선율 → 피아노 한 음
  - 섹션 전환 시 crossfade (Web Audio API)
  - 우상단 음소거 토글 (접근성)

## 4. UX 흐름 개선 (Flow Improvements)

기존 디자인 유지, 인터랙션 레이어만 추가:

### 4-1. 스크롤 인디케이터
- 우측 최소한의 점(progress dots) 12개 → 14개 (슬라이드 추가 반영)
- 현재 위치 강조, 클릭으로 점프 가능 (persona 3 'move quickly' 니즈)

### 4-2. Next 버튼 감정 차등
- 가벼운 섹션: 현재처럼 "Next" 명확
- 감정 무거운 섹션(전·결): **텍스트 없는 화살표**로 변경, 호흡 유도
- Video 섹션: onEnded 시 3초 블랙 → 자동 진행 (버튼 숨김)

### 4-3. 키보드/스와이프 지원
- ↓ ↑ / PageDown/Up → 섹션 이동
- 모바일: 스와이프 네이티브 지원 (IntersectionObserver + scroll-snap-type)

## 5. 기술 구현 범위 (Scope)

### 5-1. 건들지 않는 것 (기존 디자인 유지)
- App.css 전역 스타일
- Pretendard 폰트 체계
- 흑백 기조 색상
- 기존 컴포넌트(`Intro`, `InstagramScene`, `CatFeedingGame`, `VideoScene`, `FinalSection`)
- 이미지 자산 명명 규칙

### 5-2. 새로 만드는 것
- `src/pages/ContextBridge.js` + `.css` (S1)
- `src/pages/FirstMeeting.js` + `.css` (S2)
- `src/pages/VillageNotice.js` + `.css` (S3)
- `src/pages/MetaphorBridge.js` + `.css` (S4)
- `src/pages/MemoryWall.js` + `.css` (S5)
- `src/components/AudioLayer.js` (S6 BGM)
- `src/components/ProgressDots.js` (4-1)
- `src/hooks/useKeyboardNav.js` (4-3)

### 5-3. 수정하는 것 (최소 침습)
- `App.js`: 신규 섹션 ref/route 추가, 스크롤 네비 핸들러 확장
- `VideoScene.js`: onEnded 3초 블랙 fade 옵션
- `FinalSection.js`: 제출 후 S5(MemoryWall)로 전환 트리거
- `index.css`: 스크롤 progress용 CSS 변수 1~2개만 추가

### 5-4. 성능 개선 (곁가지)
- `<img loading="lazy" decoding="async">` 전면 적용
- `<video preload="none">` + IntersectionObserver 로 실제 화면 근접시 preload="metadata"로 승격
- 큰 PNG(평균 3~7MB) → public/assets/optimized/ 에 WebP 병행본 생성 후 `<picture>` 태그로 fallback
- 현재 총 용량: 영상 약 240MB + 이미지 약 100MB → 첫 화면 로드시 딜레이 지점 최적화
